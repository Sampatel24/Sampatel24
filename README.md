<br/>

Autonomous remediation agent that eliminates manual intervention for common Kubernetes failure modes — `CrashLoopBackOff`, `OOMKilled`, `ImagePullBackOff` — by pairing an LLM reasoning layer with a durable orchestration engine.

| Attribute | Detail |
|---|---|
| **Stack** | Claude Sonnet · Temporal · Python · Kubernetes · EKS |
| **Architecture** | 3-stage pipeline — Scan → Diagnose → Fix |
| **Reliability** | Crash-proof, retryable activity model via Temporal |
| **Observability** | Full workflow visibility through Temporal UI |
| **Impact** | Fully automated cluster self-healing with zero manual intervention |
| **Repository** | [github.com/Sampatel24/K8sSentinel](https://github.com/Sampatel24) |

K8sSentinel continuously scans cluster state, uses an LLM to generate root-cause analysis for unhealthy pods, and autonomously executes targeted fixes such as image patches and resource limit adjustments — reducing MTTR without paging an engineer.
